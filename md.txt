# Техническое задание

## Мобильное приложение Family Earn

---

# 1. Общая информация

## 1.1. Название проекта

**Family Earn**

## 1.2. Тип продукта

Кроссплатформенное мобильное приложение (iOS, Android) + веб-интерфейс для родителя

## 1.3. Концепция

Приложение для формирования у детей финансовой дисциплины через выполнение задач с вознаграждением, контролируемое родителями.

---

# 2. Цели проекта

## 2.1. Бизнес-цели

* Запуск MVP с возможностью масштабирования
* Монетизация через:
  * подписку (premium-функции)
  * in-app покупки
* Удержание пользователей через геймификацию

## 2.2. Пользовательские цели

### Родитель

* Быстро создавать задачи
* Контролировать выполнение
* Управлять финансами ребёнка
* Анализировать прогресс

### Ребёнок

* Выполнять задачи
* Получать вознаграждение
* Тратить накопленное
* Обучаться базовой финансовой грамотности

---

# 3. Роли и права доступа

## 3.1. Родитель (Admin)

* Полный доступ ко всем данным
* CRUD операций:
  * дети
  * задачи
  * награды
* Подтверждение действий ребёнка
* Управление лимитами

## 3.2. Ребёнок (User)

* Просмотр задач
* Отметка выполнения
* Просмотр баланса
* Запрос на покупку награды
* Ограниченный доступ (без редактирования)

---

# 4. Функциональные требования

---

## 4.1. Управление аккаунтом

### Регистрация

* Email + пароль
* Подтверждение email
* Выбор роли (родитель)

### Авторизация

* JWT / OAuth 2.0
* Возможность восстановления пароля

### Связь родитель–ребёнок

* Инвайт через:
  * QR-код
  * цифровой код (6–8 символов)

---

## 4.2. Управление детьми

### Функции

* Создание профиля:
  * имя
  * аватар (preset или upload)
  * возрастная категория:
    * 5–7
    * 8–14
* Поддержка нескольких детей
* Изолированные:
  * балансы
  * задачи
  * история

---

## 4.3. Управление задачами

### Создание задачи

Поля:

* title (string)
* description (text)
* reward (number)
* deadline (datetime)
* type:
  * one-time
  * recurring
* priority (optional):
  * high / medium / low
* proof_required (boolean)
* attachments (image/video)

### Типы задач

Обычная

* Выполняется один раз

Регулярная

* Настройки:
  * ежедневно
  * по дням недели
  * еженедельно

### Подтверждение выполнения

* manual (родитель)
* auto (если включено):
  * без подтверждения
  * по фото (будущий этап)

---

## 4.4. Выполнение задач (ребёнок)

* Кнопка: **"Выполнено"**
* Возможности:
  * отправка фото
  * комментарий
* Статусы:
  * pending
  * approved
  * rejected

---

## 4.5. Баланс и транзакции

### Баланс

* Отображение:
  * числовое
  * визуальное (монеты/звёзды)

### Транзакции

Типы:

* earning
* spending
* adjustment (родитель)

Поля:

* amount
* type
* timestamp
* reference_id (task/reward)

---

## 4.6. Магазин наград

### Категории

* Внутренние
* Реальные

### Создание награды

Поля:

* title
* description
* price
* category
* requires_approval (boolean)

### Покупка

* Списание баланса
* Логика:
  * instant (если digital)
  * request (если physical)

---

## 4.7. Ограничения и лимиты

* Лимит трат:
  * daily
  * weekly
  * monthly
* Ограничения:
  * по времени (например после 21:00)
  * при наличии невыполненных задач

---

## 4.8. Уведомления

### Push

Ребёнок:

* новая задача
* дедлайн

Родитель:

* задача выполнена
* запрос на покупку

---

## 4.9. Финансовая аналитика (родитель)

* Метрики:
  * заработано
  * потрачено
  * баланс
* Периоды:
  * день / неделя / месяц
* Экспорт:
  * CSV
  * PDF

---

## 4.10. История

* Лента событий:
  * выполнение задач
  * покупки
* Фильтры:
  * по дате
  * по типу

---

## 4.11. Кастомизация (8–14 лет)

* Темы интерфейса
* Аватары
* Цвет баланса
* Таблица доходов/расходов

---

# 5. UX/UI требования

## 5.1. Общие принципы

* Минимализм
* Крупные элементы
* Интуитивная навигация
* Геймификация

* Анимации

### 8–14 лет

* Полный интерфейс
* Финансовая аналитика

---

## 5.3. Основные экраны

### Родитель

* Dashboard (дети + баланс)
* Создание задачи
* Магазин наград
* Отчёты

### Ребёнок

* Мои задачи
* Магазин
* История

# 7. Безопасность

* JWT авторизация
* Role-based access control (RBAC)
* Ограничение прав ребёнка
* Защита API (rate limiting)

### Работа с деньгами

* На MVP:
  * только виртуальная валюта
* Реальные деньги:
  * только через подтверждение родителя
  * без прямых транзакций (во избежание лицензирования)

---

# 8. Интеграции (будущее)

* Платёжные системы
* Банковские API
* AI-рекомендации
* Голосовой ввод

---

# 9. Нефункциональные требования

* Производительность:
  * отклик < 300ms
* Масштабируемость:
  * поддержка 10k+ пользователей
* Надёжность:
  * uptime 99.5%
* Безопасность:
  * соответствие GDPR / COPPA

---

# 10. User Stories (ключевые)

* Родитель создаёт задачу → ребёнок выполняет → родитель подтверждает → начисление
* Ребёнок покупает награду → родитель подтверждает
* Родитель устанавливает лимиты → система ограничивает действия

---

# 11. План разработки

## Этап 1 (MVP) — 2–3 месяца

* Регистрация
* Инвайт ребёнка
* Простые задачи
* Баланс
* История
* Базовый магазин

## Этап 2 — 3–4 месяца

* Регулярные задачи
* Фото-подтверждение
* Аналитика

## Этап 3 — 2–3 месяца

* Интеграции
* AI
* Групповые задачи

---

# 12. KPI

* DAU / MAU
* % выполненных задач
* Retention (7/30 дней)
* Среднее количество задач на пользователя
* Частота использования (цель: 5+ раз в неделю)

---

# 13. Риски и ограничения

| Риск | Решение |
| ---- | ------- |
| Фейковое выполнение задач | Фото / подтверждение |
| Конфликты из-за денег | Только виртуальная валюта на старте |
| Непонимание интерфейса | Геймификация и визуал |
| Юридические риски | Минимизация данных, GDPR |
